Step 03 — Intuition
Why the PR Curve looks like stairs
Three key patterns explain the characteristic shape of every PR Curve — and what they tell you about the model.
① TP hit → step right (Recall ↑)
When a new detection is a TP, the cumulative TP count goes up. Recall = TP / GT_total, so Recall increases. The curve steps right.
Recall = TP↑ / GT_total → moves right
② FP hit → step down (Precision ↓)
When a new detection is a FP, TP stays the same but TP+FP grows. Precision = TP / (TP+FP) drops. The curve steps down.
Precision = TP / (TP + FP↑) → moves down
③ High-conf detections are added first
Because we sort by confidence descending, the model's most certain predictions come first — these tend to be correct, so the curve starts high-precision. It sags as we add lower-confidence (noisier) detections.
Sort: conf 0.97 → 0.93 → … → 0.06
Takeaway: A curve that stays top-right means the model finds many TPs early (high precision) and continues finding them (high recall). AP is the area under this curve — bigger = better.

11-point (VOC 2007)
Samples recall at exactly 11 fixed levels: 0.0, 0.1, …, 1.0. At each level, takes the max precision to the right. Then averages the 11 values.
AP = (1/11) Σ P_max(r)
r ∈ {0, 0.1, 0.2 … 1.0}
Problem: if the curve is sparse between two sample points, the area estimate can be wrong — it either over- or under-counts.
All-point (VOC 2010+)
Uses every recall point where the curve actually changes. Computes width × height for each tiny rectangle and sums them — a true Riemann integral.
AP = Σ (r_n − r_{n-1}) × P_max(r_n)
for all actual change points
Result: exactly the area under the monotone-envelope curve, no information is missed or double-counted.
Key insight: When the PR curve drops sharply between two 11-pt sample points, 11-point misses that drop. All-point catches it. That's why the same model can look better under 11-point.
Step 04 — Envelope
Raw Curve → Monotone Envelope → Area
Before computing AP, both methods apply the same envelope step. Watch the three stages one by one.
Stage 1 of 3
Raw PR Curve
This is the curve you get by sorting detections by confidence and plotting (Recall, Precision) at each step.

It has a staircase shape — right steps from TP hits, downward steps from FP hits.
Each point = one detection added
TP → right   FP → down
Stage 2 of 3
Monotone Envelope
Scan from right to left. At each recall point, replace precision with the maximum precision achievable at that recall or higher.

This removes the "dips" — a dip just means a slightly higher confidence threshold would recover that precision.
P_env(r) = max P(r') where r' ≥ r
Orange line = envelope. Note how it never dips below the maximum it has seen to its right.
Stage 3 of 3
Area = AP
AP is the area under the envelope, not the raw curve.

The step-downs in the raw curve are gone — replaced by the envelope. This is why AP doesn't penalise individual FPs harshly; it asks: "what's the best precision this model can achieve at each recall level?"
AP = ∫ P_env(r) dr
= area of the green region

Key Concepts at a Glance
The full 5-step pipeline on one slide.
01
IoU
Ratio of intersection to union area between two boxes. PASCAL VOC uses 0.5 as the threshold to decide TP vs FP.
IoU = |A ∩ B| / |A ∪ B|
02
TP / FP / FN
IoU ≥ thr → TP. Second match on same GT → FP. Undetected GT → FN. TN is undefined — no background class.
Precision = TP/(TP+FP) Recall = TP/(TP+FN)

AP = ∫₀¹ P(r) dr

Arithmetic mean of per-class AP over all 20 classes. VOC uses mAP@0.5; COCO uses mAP@0.5:0.95 (10 IoU levels averaged).
mAP = (1/C) × Σ AP_c
FP Deep Dive
The 3 Types of False Positives
Any detection labelled Positive that does not count as TP falls into one of three categories.
Case 1 — IoU Miss
Box was predicted but barely overlaps GT. The location is simply wrong.
IoU < threshold → FP
Case 2 — Duplicate Detection
IoU is fine, but another prediction already claimed this GT. NMS prevents this.
1st match → TP, rest → FP
Case 3 — Hallucination
A box drawn in empty background with no GT nearby. A model hallucination.
No GT in region → FP
